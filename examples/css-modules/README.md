# CSS Modules Example

The same info card as [`css-global`](../css-global), but styled through a locally scoped
`*.module.css` file instead of a global stylesheet.

The card is also rebuilt in [`css-preprocessors`](../css-preprocessors), [`css-postcss`](../css-postcss)
and [`tailwindcss`](../tailwindcss), so the five examples can be compared side by side —
same UI, different styling pipeline.

## Features Exercised

- `*.module.css` recognition rules — a file is treated as a CSS Module by its name
- Local scope: class names are hashed at build time and reached through the imported object
- Rendering the generated class name at runtime (`{styles.card}`), so the hash is visible on screen
- Type declarations generated by [`@rsbuild/plugin-typed-css-modules`](https://rsbuild.rs/plugins/list/plugin-typed-css-modules),
  checked in as `src/App.module.css.d.ts` so the example typechecks without a build

## Getting Started

```bash
pnpm install
pnpm run dev
```

Scan the QRCode in the terminal with your LynxExplorer App to see the result.
